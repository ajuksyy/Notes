Relational Data Models

Repersents data in the form of tables. THis consists of 3 components

  • → Data structure: orgnise in the form of tables
  • → Data manipulation: Powerful operations used to manipulate data
  • → Data integrity: Used to specify business rules to specify to maintain integrity during manipulation

Objectives of relational Data Model:

→ High degree of data independence

→ Provide substantial grounds for dealing with data sematics, consistency and reduncy problems

→ Enable the expansion of set-oriented data manupilation languages

→ If there is on instance, it's a populated table

→ Connection between math concept relation and this:-

→ * Use of set theory & predicate logic principles to define & momupilate dates in relational database

→ * This provides powerful and expressive framerwork for representing query, and managinng data in a structural and systematic manner

Properties:

→ Distinct Name

→ Each cell of the relation contains 1 value

→ Values of an attri, are all from some domain

→ Order of attribute has no significance

Key

→ Super: Attribute, that uniquely identifies a tuple within a relation

→ Candidate: min, set of attributes that uniquely identifies each tuple of a relation

→ Alternate: not seleced to be primary

→ Primary: selected to identify tuples uniquely with the relation. Usually underlined in the schema

→ Composite: consists with one or more

→ Foreign: Attribute, within one relation that matches the candidate key to another

integrity constraint

→ Ensure data is accurate, facilitate maintaining the accuracy and integrity in the database

Types

→ Domain constraints

→ Entity integrity

→ referential integrity

Domain constraint: set of values assigned to an attribute

→ Consists of domain name, Meaning, Data type, Size, and allowable values or range

Entity integrity: Ensure that every relation has a primary key, all values are valid. No primary key can be null

Referential integrity: requires that a Foreign key must have a matching primary key or null.

→ reference from a row in one table to another table must be valid

→ orphan record: foreign key value is not found in the corresponding entity

General constraint: additional rules specified by user

Purpose of views

→ Powerful and flexible security mechanism by hiding

→ Permits users to access data customised to that user

→ Simplify complex operation

Reductant data: When some piece of data exists in multiple places

degree No. of entities that have taken part in the relationship

→ Physical: Define how its actually stored

→ Logical: Relationship which exists among stored data
